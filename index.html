팀원 소개 & 역할 분담

2조 퍼피코더스👣입니다.
"퍼피(Puppy)"는 영어로 강아지를 뜻하며, "코더스(Coders)"는 개발자를 의미합니다. "강아지 코더들"을 의미하는 뜻으로 "함께하개" 주제와 맞는 팀명을 짓게 되었습니다.

기능 소개

1. 반응형 웹 페이지
  • 반응형 웹 페이지를 만들기 위해 Bootstrap을 사용했습니다.
  • 데스크톱 컴퓨터, 테블릿, 스마트폰 등 다양한 환경에서 이용가능 합니다.
2. 로그인 페이지
  • ID와 비밀번호 입력 없이 로그인 시도 시 입력해야 한다는 메시지가 표시됩니다.
  • 아이디와 비밀번호가 일치하지 않을 경우에도 메시지가 표시됩니다.
3. 회원가입 페이지
  • 아이디는 이메일로 회원가입이 되도록 했습니다.
  • 중복 확인 기능을 넣어 기존에 동일한 ID가 있으면 가입이 되지 않도록 했습니다.
  • 개인정보처리방침에 체크해야 회원가입이 완료됩니다.
4. 반려동반 문화 시설 검색
  • 시도, 시군구, 동읍면 을 선택하면 그 지역의 병원 목록이 하단에 나타납니다.
  • 해당 병원을 클릭하면 지도에 위치가 표시되고 업체 정보가 업체 사진과 함께 상단에 나타납니다.
5. 다이어리 페이지
  • 반려견에 대한 다이어리를 작성할 수 있습니다.
6. 펫 프로필 등록 페이지
  • 반려견에 대한 프로필을 등록 할 수 있습니다.
  • 파일을 선택하여 반려견 이미지도 등록 가능 합니다.
7. 내정보 페이지
  • 로그인된 id 정보를 불러와 작성한글, 문의내역, 다이어리에 내용을 표시해 줍니다.
  • 포인트, 쿠폰 목록은 기능 구현 되지 않았습니다.
8. 공지사항 페이지
  • 회원은 글 등록 버튼이 없습니다.
  • 관리자 ID로 접속해야 공지글을 작성할 수 있습니다.
9. 게시판 글작성
  • 게시판 글을 작성하려면 로그인이 필요합니다.
  • 글 작성 페이지로 이동하면 자동으로 로그인 정보가 불러와집니다.
10. 게시판 댓글등록
  • 로그인한 아이디로 댓글을 등록할 수 있습니다.
  • 댓글 목록에는 작성자 이름과 날짜가 표시됩니다.
11. 검색기능
  • 공지사항, 게시판, 그리고 Q&A 페이지에서 각각 게시글을 검색할 수 있습니다.
  • 제목, 내용, 작성자로 검색이 가능합니다.
12. Q&A 글작성
  • Q&A 글을 작성하려면 로그인이 필요합니다.
  • Q&A 글 작성 페이지로 이동하면 자동으로 로그인 정보가 불러와집니다.
  • 비밀글로 등록하려면 비밀글 체크박스를 선택할 수 있습니다.
  • 비밀글 체크박스를 선택하면 비밀번호란이 활성화 됩니다.
  • 비밀번호 없이 글등록 시 메시지가 발생합니다.
  • 개인정보처리방침에 동의해야 글 등록이 가능합니다.
13. Q&A 답변등록 상태 변경
  • Q&A글에 대한 답변은 관리자만 가능합니다.
  • 답변이 완료 된 후에는 미답변에서 답변완료로 글 상태가변경됩니다.
    • 14. Q&A 비밀글
    • 비밀글로 된 글을 조회할 경우 비밀번호 입력창이 나타납니다.
    • 비밀글 작성자 ID와 현재 로그인된 ID가 동일할 시에만 나타납니다.
    • 동일하지 않을 경우 "비공개글 입니다" 메시지가 표시됩니다.

    • 펫을 등록한 후에도 수정할 수 있는 기능을 추가합니다.
    • 등록한 펫으로 다이어리를 작성할 수 있도록 설정합니다.
    • 캘린더 기능을 추가하여 건강 검진일을 손쉽게 추가할 수 있도록 구현합니다.

    느낀점

    첫 수업을 들었을 때가 엊그제 같은데, 벌써 6개월이라는 시간이 지나 팀프로젝트를 마무리하기까지가 아직도 믿기지 않습니다.
    개발자로 걸어가기 위해 준비하고 팀프로젝트를 진행하며, 처음으로 깃 사용법을 배우고 Notion을 활용해 보는 등 많은 새로움이 있었습니다. 부족한 점도 많았지만, 묵묵히 따라와 주는 팀원들에게 큰 감사를 전합니다.

    기능

    1. 반응형 웹 페이지
    • 반응형 웹 페이지를 만들기 위해 Bootstrap을 사용했습니다.
    • 데스크톱 컴퓨터, 테블릿, 스마트폰 등 다양한 환경에서 이용가능 합니다.
    2. Swiper 사용
    • 인기영화의 Top5만 나타나게 하였습니다.
    • Swiper의 종류 중 Coverflow Effect를 사용했습니다.
    • 마우스를 올리면 슬라이드가 멈추도록 설정되어 있습니다.
    • 영화 포스터 위에 영화 제목을 표시하고, 포스터가 흰 배경인 경우를 고려하여 text에 shadow 효과를 추가했습니다.
    • 메인 페이지에는 Swiper Pagination을 사용하여 마우스를 올리면 화살표 버튼이 나타납니다.
    • 영화 상세 페이지에는 Swiper Scrollbar를 사용하여 마우스 드래그를 통해 슬라이드를 이동할 수 있도록 했습니다.
    3. 영화 순위 보여주기
    • forEach의 index를 사용하여 영화 poster card 상단에 순위가 표시되도록 했습니다.
    4. 영화 카드 MouseOver 효과
    • 영화 poster 이미지가 없는 경우 맨왼쪽 이미지와 같이 기본 이미지가 나타납니다.
    • 영화 poster card에 마우스를 올리면 해당 영화의 별점과 간단한 설명이 나타납니다.
    • 설명이 길 경우를 고려해 말줄임표로 표시했습니다.
    5. 별점 표시, 공유 기능
    • 영화 제목 상단에 해당 영화의 평점을 불러와 별 모양으로 표시하도록 구현했습니다.
      • 평점은 10점 만점의 값으로 되어 있기 때문에 평점을 2로 나눈 값으로 표시했습니다.
    • window.navigator.share()를 이용하여 공유 버튼을 클릭하면 해당 주소를 공유할 수 있도록 했습니다.
    6. 더보기 기능
    • 영화에 대한 소개글이 길 경우 css를 이용해 말 줄임표로 표시하고, "더보기" 버튼을 클릭하면 전체 내용이 나타나도록 구현했습니다.
    7. 트레일러 영상 보기
    • 해당 영화의 트레일러 영상을 영화 상세 설명 하단에 바로 시청할 수 있도록 구현했습니다.
    • 트레일러 영상이 없을 경우에는 "영상이 아직 등록되지 않았습니다" 메시지가 나타납니다.
    8. 위로 scroll 기능
    • 페이지 오른쪽 하단에 보이는 위로 화살표 모양을 클릭하면 페이지 상단으로 이동합니다.
    9. 최근 검색어 저장
    • 이전에 검색된 내용이 없으면 "검색 내역이 없습니다"라고 표시됩니다.
    • 검색할 때마다 localStorage에 검색 값이 배열로 저장됩니다.
    • 검색내용을 개별적으로 삭제하거나 모두 한 번에 삭제할 수 있습니다.

    공유하기 기능 구현하기
    • "공유하기" 버튼을 클릭했을 때, 사용자의 브라우저가 공유 기능을 지원 하는 경우 해당 영화 정보를 공유할 수 있도록 구현했습니다.
    • window.navigator.share()메서드를 사용하여 웹 플랫폼의 공유 기능을 호출합니다. 이 메서드는 사용자가 선택한 공유 대상에 따라 텍스트, 제목, url을 포함하여 데이터를 공유합니다.
    • url에는 window.location.href로 설정하여 현재 페이지를 공유 url로 설정했습니다.
    별점 표시 기능 구현하기
    • 영화 데이터에서 받아온 vote_average(평점) 값을 기반으로 별점을 시각적으로 표현하는 HTML 문자열을 생성하는 코드 입니다.
      • vote_average : 영화의 평점을 10점 만점에서 받아왔으며, 이를 5점 만점으로 변환하기 위해 2로 나누어vote_average에 저장합니다.
      • vote_average_remain : 반별을 표시하기 위해vote_average에서 정수 부분을 뺀 나머지 값을 계산 했습니다.
    • 반복문을 통해 총 5개의 별 아이콘을 생성합니다.
      • Math.floor(vote_average) : vote_average의 정수 부분만큼 꽉찬별 아이콘을 추가합니다.
      • vote_average_remain >= 0.5 : 평점 0.5 이상인 경우 반복문의 현재 위치에 반별 아이콘을 추가합니다.
      • 그 외의 경우에는 빈별 아이콘을 추가합니다.

    검색버튼 event시 값 전달

    • e.preventDefault(); : 이벤트 객체(e)의 기본 동작을 취소합니다. form이 submit됐을 때 페이지가 리로드되는 동작을 막습니다.
    • initMovieSearchPage(searchValue); : initMovieSearchPage함수를 호출하여 검색된 영화의 결과를 표시하도록 했습니다.

    • 사용자가 입력한 검색어를 최근 검색어 목록에 추가하는 기능을 수행합니다.

    • 새로운 검색어 객체(newKeyword)를 기존의 검색어 배열(keywords)의 앞에 추가하여, 업데이트 된 최근 검색어 목록을 만듭니다.
    • JSON.stringify()를 사용하여 배열을 JSON문자열로 변환하여 localStorage에 저장합니다.>
    • 이렇게 함으로써 최근 검색어 목록이 계속해서 유지되며, 다음 검색 시에도 이 목록에 새로운 검색어가 추가 됩니다.

    • 검색어 목록이 없을 경우를 대비해 HTML에 안내문구를 표시하기 위해서if (keywordsList.length > 0)을 사용하였습니다.
    • onclick="deleteKeyword(${element.id})" : 해당 검색어 삭제를 위해 추가했습니다.
    최근 검색어 모두 삭제
    • localStorage.clear();: locaStorage에서 모든 데이터를 삭제합니다.
    • getKeyword(); : 모든 검색어 초기화 후 HTML 화면을 update하기 위해 함수를 다시 불러옵니다.
    최근 검색어 단일 삭제
